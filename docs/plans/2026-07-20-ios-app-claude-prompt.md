# YomiagePlayer iOS版 — Claude Code 実装プロンプト

> このドキュメントは、Windows版 YomiagePlayer (WPF + Whisper.net + LibVLC) と同等の機能を持つ
> iOSアプリを Claude Code (Sonnet) に実装させるためのプロンプト兼仕様書。
> **新規リポジトリで単体で読める**ように、移植対象のアルゴリズムは本文に完全な仕様として記載してある。
> Windows版のソースは参照できない前提で書かれている。
>
> タスク単位の実行手順は同日付の `2026-07-20-ios-app-implementation-plan.md` にある。
> 実装を始めるときは、本書を読んだ上で実装計画のタスクを番号順に実行すること。

## あなた(Claude Code)への指示

あなたはこれから、音声・動画ファイルを再生しながら **端末内(オンデバイス)の Whisper 音声認識**で
文字起こしを行い、カラオケの歌詞のように再生位置と同期表示する iPhone アプリを実装する。

- 言語/UI: **Swift + SwiftUI**、ターゲット **iOS 17 以上**、iPhone 縦画面を主対象
- アーキテクチャ: MVVM。ロジックは UI 非依存の Swift Package(`YomiageCore`)に分離し、単体テスト可能にする
- 音声認識: **WhisperKit**(CoreML、Swift Package)を使う。モデルはアプリに同梱せず初回に設定画面からダウンロード
- 再生: **AVFoundation (AVPlayer / AVAudioSession)**。バックグラウンド再生・ロック画面操作対応
- テスト: Swift Testing (または XCTest)。Core のロジック(ハッシュ・フィルタ・キャッシュ・セグメント検索・
  アートワーク探索・m3u8)は必ずテストを書いてから実装する
- 実装は下の「マイルストーン」の順に進め、各マイルストーンごとにビルドとテストが通る状態を保つ

### 想定ユースケース(重要)

主用途は **ASMR・ボイス作品・長時間の音声コンテンツ**。つまり:

- 1ファイルが数十分〜数時間と長い。文字起こしは**ストリーミング表示**(全部終わるのを待たず、
  認識できたセグメントから順次表示)が必須
- 無音・環境音・非音声区間が多く、Whisper が**幻覚(ハルシネーション)**を起こしやすい。
  後述のテキストベースフィルタを必ず実装する
- 同じファイルを繰り返し聴くので、文字起こし結果は**コンテンツハッシュでキャッシュ**し、
  2回目以降は即表示する(リネーム・移動してもキャッシュが効くこと)

## 画面構成 (iPhone)

Apple Music 型の構成にする:

```
┌─────────────────────────┐
│  (各タブのコンテンツ)      │
│                         │
│                         │
├─────────────────────────┤
│ ▶ ミニプレイヤーバー       │  ← 再生中のみ表示。タップで再生画面をフルスクリーン表示
├─────────────────────────┤
│ ライブラリ│プレイリスト│設定 │  ← TabView
└─────────────────────────┘
```

### 1. ライブラリタブ
- ユーザーがフォルダを登録する(Files アプリのフォルダを `UIDocumentPickerViewController` で選択、
  **security-scoped bookmark** として永続化。次回起動時は bookmark を解決してアクセス)
- 登録フォルダの一覧 + 各フォルダを開くと配下のメディアファイルを再帰列挙して表示
  (サブフォルダごとにグルーピング表示)
- ファイル名によるインクリメンタル検索(登録フォルダ全体を横断)
- 行タップ → そのフォルダの全ファイルをプレイリストに置き換えて、タップした曲から再生
- スワイプ/長押しメニュー: 「プレイリストに追加」「フォルダを登録解除」
- 各行にサムネイル(後述のアートワーク探索)を表示

### 2. プレイリストタブ
- 現在の再生キュー。ドラッグで並べ替え、スワイプで削除
- フォルダごとのセクションヘッダ表示(同一フォルダの曲をまとめる)
- m3u8 形式でエクスポート/インポート(Files 経由の share / import)
- 現在再生中の行をハイライト

### 3. 再生画面(フルスクリーン、ミニプレイヤーから遷移)
- 上部: アートワーク(横スワイプで複数候補を切り替えられるギャラリー)。動画ファイルなら映像を表示
- 中央〜下部: **文字起こしトランスクリプト**(タイムスタンプ付きセグメントのリスト)
  - 再生位置に合う行を自動ハイライト+自動スクロール
  - ユーザーが手でスクロールしたら自動スクロールを一時停止し、「現在位置に戻る」ボタンを出す
  - 行タップでその時刻にシーク
  - 解析中は末尾に進行状況を表示。再生位置が解析済み範囲を追い越したら
    「この部分を解析中…」バナーを出す(何も出ないより明示的に)
  - 「再解析」ボタン(キャッシュ破棄して解析し直し)
- 下部: シークバー、再生/一時停止、前/次トラック、音量、再生速度(0.5x–2.0x)
- ロック画面/コントロールセンター: `MPNowPlayingInfoCenter` + `MPRemoteCommandCenter` で
  タイトル・アートワーク・シーク・前後トラックを提供

### 4. 設定タブ
- Whisper モデル選択とダウンロード管理(進捗表示、削除):
  - 推奨既定値: `small`(iPhoneでは速度・発熱・電池のバランス優先。Windows版の既定は medium だが
    モバイルでは small を既定にする)
  - 選択肢: `tiny` / `small` / `medium`(large系は iPhone のメモリ・発熱的に対象外とする)
- 認識言語(既定: 日本語)
- アイドル時の先読み解析の ON/OFF(既定 ON)+「充電中のみ」オプション(既定 ON)
- キャッシュ使用量表示とクリア
- OSSライセンス表示

## Core 仕様(Windows版から移植するロジック)

以下は Windows版で実証済みの設計。**意図的な決定なので変えないこと。**

### コンテンツハッシュ (`ContentHasher`)
文字起こしキャッシュのキー。**フルファイルのハッシュではない**:

```
key = SHA256( fileSize(8バイトLE) + 先頭1MB + 末尾1MB ) の hex
```

- ファイルが 2MB 未満なら全体を読む
- 理由: 数GBの動画でも一瞬で計算でき、リネーム・移動してもキャッシュが生き残る

### 文字起こしキャッシュ (`TranscriptionCache`)
- 保存先: アプリの `Application Support/cache/` 配下、ファイル名 `{hash}-{model}.json`
- 内容: `{ language, segments: [{ start, end, text }] }`(秒単位の Double)
- **書き込みは原子的に**: `.tmp` に書いてから rename(長時間解析の結果を壊さないため)
- ロード失敗(不存在・破損JSON)は「キャッシュなし」として扱いクラッシュしない

### 解析キュー (`TranscriptionQueue`)
- **同時実行は常に1ジョブ**。待ち行列は **LIFO**(新しく開いた曲が最優先)
- ユーザー起因のジョブ(曲を開いた/再解析)は `preempt: true` で投入し、
  **実行中のジョブをキャンセルして即座に割り込む**。中断されたジョブの途中結果は破棄(キャッシュしない)
- バックグラウンド先読みジョブは `preempt: false`。互いに割り込まないし、ユーザージョブにも割り込まない
- Swift では actor + `Task` のキャンセルで実装(構造化並行性)

### 解析コーディネータ (`TranscriptionCoordinator`)
曲が変わったときの流れ:

1. ハッシュ計算 → キャッシュヒットなら即トランスクリプト表示、終了
2. ミス → キューに preempt 投入。ジョブ内容:
   a. AVAssetReader で **16kHz モノラル Float32 PCM** にデコード(WhisperKit の AudioProcessor が
      ファイルURL直接入力に対応していればそれで良い。動画からの音声抽出も AVAssetReader で行う。
      キャンセル時はデコード・認識とも即中断)
   b. WhisperKit でセグメント単位のストリーミング認識(セグメント確定ごとにコールバック)
   c. 各セグメントを **ハルシネーションフィルタ**に通し、通過したものだけ UI に追加(逐次表示)
   d. 完走したらキャッシュに原子的保存。キャンセルされたら保存しない
3. **UI 更新は「現在表示中の曲のキーと一致するか」でゲート**する。裏で走っていた古いジョブの
   完了イベントが、いま見ている曲の表示を上書きしてはならない

### ハルシネーションフィルタ (`HallucinationFilter`)
セグメントを捨てるルール(順に判定):

1. `（...）` `(...)` `[...]` `【...】` の括弧書き注釈を除去した結果、
   **何も残らなければ捨てる**(例: `(笑)` 単体は捨てる。`そうなんだ(笑)` は残す)
2. 正規化(空白・句読点・記号を除去)したテキストが既知の定型句で**始まる**なら捨てる:
   ```
   ご視聴ありがとうございました / ご清聴ありがとうございました / チャンネル登録 /
   最後までご視聴 / おやすみなさい。おやすみなさい。 / 字幕視聴ありがとうございました
   ```
   (このリストは実サンプル検証で追加していく前提の設計にする)
3. **2秒未満**のセグメントで、直前セグメントと正規化テキストが完全一致なら捨てる
   (繰り返しループ幻覚)。2秒以上の反復は歌のサビ等の可能性があるので残す

加えて WhisperKit 側のオプションで `noSpeechThreshold` / 温度フォールバック等、
無音対策に使えるパラメータがあれば設定する(コンテキスト持ち越しは無効化)。

### 再生位置→セグメント検索 (`SegmentLocator`)
- `start` 昇順のセグメント配列に対する**二分探索**
- 区間は半開 `[start, end)`。**セグメント間の空白時間では -1(ハイライトなし)を返す**
  (直前の行にハイライトが残り続けてはいけない)
- 「再生位置が解析済みの最後のセグメントより先にあり、かつ解析継続中」を検出するフラグを提供
  (UI の「この部分を解析中」バナー用)

### アイドル先読み (`IdleAnalysisService`)
- 定期タイマーでキューが空(アイドル)のときだけ動く
- 優先順: **① 現在のプレイリストの「これから再生される曲」をリスト順** → ② 登録フォルダ全体を走査し、
  未キャッシュのファイルを 1 tick に 1 件だけ解析
- 結果はキャッシュに入れるだけで、表示中のトランスクリプトには一切触らない
- セッション内 visited セットで再走査を防止。フォルダ登録が変わったらリセット
- **iOS 固有の制約**: フォアグラウンドでのみ動かす。設定で「充電中のみ」(既定ON)を尊重し、
  `ProcessInfo.processInfo.thermalState` が `.serious` 以上なら停止する。
  再生中の曲の解析(ユーザージョブ)はバックグラウンド再生中も継続してよいが、
  先読みはバックグラウンドでは行わない

### アートワーク探索 (`ArtworkLocator`)
表示優先順:

1. **メタデータ埋め込み画像**(`AVAsset` の metadata / artwork)。複数あれば全部候補にする。
   ただし壊れた・空のデータ(数バイトのゴミ)は除外し、除外の結果ゼロならフォルダ画像へフォールバック
2. フォルダ画像(拡張子: jpg/jpeg/png/webp/bmp/gif):
   - メディアと同じフォルダ → なければ、そのファイルが属する**ライブラリ登録フォルダ配下を再帰的に**
     全画像候補として採用(登録フォルダはユーザーが明示した境界なので深掘りしてよい)
   - 登録フォルダ経由でない場合は、祖先フォルダを**最大2階層**まで遡る。
     `.meta.json` マーカーがあるフォルダは「作品フォルダ」とみなし配下を再帰探索してよい。
     マーカーがない祖先は直下(非再帰)のみ、かつ「メディアと同名 or 定番名に一致」した場合だけ採用
     (無関係な共有フォルダの画像を拾わないため)
3. 候補の並び順: **メディアと同名 → 定番名(cover, folder, front, album, jacket, artwork, thumbnail)
   → 名前順**。先頭がサムネイル、全候補は再生画面のギャラリーでスワイプ切替
- 埋め込み画像とフォルダ画像は**混在させない**(埋め込みがあればフォルダは見ない)
- サムネイルはデコード済み小サイズでメモリキャッシュ(`NSCache`)し、リストのスクロールを軽く保つ

### m3u8 (`M3u8Serializer`)
- 保存: `#EXTM3U` + 各曲 `#EXTINF:-1,{拡張子なしファイル名}` + パス、UTF-8 (BOMなし)
- 読み込み: `#` 行と空行をスキップ、相対パスは m3u8 ファイルの場所基準で解決
- iOS では bookmark 化した URL との突き合わせが必要になるので、読み込み時に
  登録フォルダ配下のパスとマッチングして解決する

### 対応フォーマット
- Windows版の対応: mp3, wav, flac, m4a, ogg, opus + 動画 mp4, mkv, avi, webm
- **iOS v1 では AVFoundation がネイティブ再生できるものに絞る**:
  音声 = mp3, wav, flac, m4a, aac / 動画 = mp4, mov, m4v
- ogg/opus/mkv/webm/avi は v1 では「非対応」としてライブラリ上でグレーアウト表示
  (将来課題として README に明記。ffmpeg 系ライブラリの組み込みは v1 ではやらない)

### 設定の永続化
- JSON 1ファイル(登録フォルダの bookmark データ、モデルID、音量、言語、先読み設定)
- 書き込みは `.tmp` → rename の原子的置換。読み込み失敗時は既定値

## Whisper モデル管理

- WhisperKit の CoreML 変換済みモデル(Hugging Face `argmaxinc/whisperkit-coreml`)を
  設定画面からダウンロード。進捗表示・キャンセル・削除・ダウンロード済みバッジ
- モデル格納先は `Application Support/models/`。**`isExcludedFromBackup = true`** を付ける
  (数百MB〜GBを iCloud バックアップに入れない)
- キャッシュファイル名に使うモデルIDは `tiny` / `small` / `medium`
- モデル未ダウンロード時に曲を開いたら、再生は普通に始めて、トランスクリプト欄に
  「モデルをダウンロードすると文字起こしが使えます」+設定への導線を表示(再生機能を人質にしない)

## マイルストーン(この順で実装すること)

各マイルストーン完了時にビルド+全テストが通ること。UIの見た目の作り込みより先に配線を通す。

1. **プロジェクト骨格 + YomiageCore パッケージ**
   - `ContentHasher` / `TranscriptionCache` / `HallucinationFilter` / `SegmentLocator` / `M3u8Serializer` /
     設定ストアを、上の仕様どおりテストファーストで実装(このマイルストーンはUIなしで完結)
2. **再生の縦切り**
   - AVPlayer ラッパー(再生/一時停止/シーク/速度/音量/曲末イベント)、AVAudioSession 設定、
     バックグラウンド再生、Now Playing / Remote Command、ミニプレイヤー+フルスクリーン再生画面(操作系のみ)
   - Files から1ファイル選んで再生できる状態にする
3. **ライブラリ + プレイリスト**
   - フォルダ登録(security-scoped bookmark 永続化)、再帰列挙、検索、プレイリスト
     (並べ替え・削除・前後トラック・自動連続再生)、m3u8 入出力
4. **文字起こしの縦切り**
   - WhisperKit 統合、キュー(LIFO+preempt)、コーディネータ(ハッシュ→キャッシュ→解析→ゲート付きUI更新)、
     トランスクリプトUI(ストリーミング追加・ハイライト・自動スクロール・タップシーク・解析中バナー・再解析)
   - モデルダウンロード画面もここで(ないと動かせないため)
5. **仕上げ**
   - アートワーク探索+ギャラリー、アイドル先読み(充電中のみ/発熱ガード)、
     キャッシュ管理画面、ライセンス表示、非対応形式のグレーアウト、細部の磨き

## テストについて

- Core のロジックは全て純粋 Swift でテストする(ファイルI/Oは一時ディレクトリ)
- フィクスチャ: 1秒サインの wav をテストターゲットに同梱。文字起こしの結合テストは
  実モデルが要るため CI では行わず、`TranscriberProtocol` のフェイクでコーディネータ/キューの
  挙動(preempt・ゲート・キャッシュ保存条件)を検証する
- 特に重点的にテストする不変条件:
  - preempt されたジョブの結果がキャッシュされないこと
  - 古いジョブの完了が現在表示中の曲の表示を上書きしないこと
  - セグメント間の空白でハイライトが -1 になること
  - フィルタの3ルール(括弧のみ/定型句/短時間反復)と「本文+括弧は残す」こと
  - キャッシュ・設定の書き込みが原子的であること(tmp 経由)

## やらないこと (v1 スコープ外)

- iPad / macOS 対応(iPhone 縦画面優先。ただし SwiftUI なので極端に壊れない程度でよい)
- ogg/opus/mkv/webm/avi の再生
- large 系モデル、翻訳、話者分離
- クラウド同期・アカウント機能(全機能オフライン・オンデバイスで完結すること)
- Windows版とのキャッシュ互換(ハッシュ仕様は同じだが、モデルが CoreML 版なので結果は別物)
