# YomiagePlayer 設計書

作成日: 2026-07-08
改訂: 2026-07-08 アーキテクチャレビュー(Fable)の指摘を反映

## 概要

Windows向けのメディアプレイヤー。ASMR・話し声・歌(BGMあり)などの音声/動画ファイルを解析し、日本語の歌詞・文章に書き起こして、再生に同期した字幕風表示を行う。一度解析した内容はキャッシュファイルに保存し、同じファイルの再解析を避ける。加えて、フォルダ登録・プレイリストなど基本的なメディアプレイヤー機能を備える。

## 技術スタック

- 実装言語/フレームワーク: C# / .NET 10 (LTS), WPF
- 再生エンジン: LibVLCSharp(VLCエンジン) — mp3/mp4/mkv/flac/wavなど幅広いフォーマットをコーデック追加不要で再生。LGPL 2.1のためライセンス表記画面を設ける
- 音声認識: Whisper.net(whisper.cppのC#バインディング)。Pythonランタイム不要でexe配布が容易
  - ランタイム: `Whisper.net.Runtime.Cuda` / `Whisper.net.Runtime.Vulkan` / CPU版を同梱し、CUDA→Vulkan→CPUの優先順で自動フォールバック
  - モデル: 設定画面で `small` / `medium` / `large-v3-turbo` を選択可能。デフォルトは medium(GPU検出時は turbo を推奨表示)
- 動画からの音声抽出・フォーマット変換: FFMpegCore + **LGPLビルドのffmpeg**(音声抽出とWAV変換のみ使用するためLGPLで十分。GPLフルビルドは使わない)。Whisper.net向けに16kHz mono WAVへ変換

## MVPスコープ

含む:
- LibVLCSharpによる音声/動画再生 + 基本コントロール(再生/一時停止/前後/シーク/音量/リピート/シャッフル)
- プレイリスト(ドラッグ&ドロップ並び替え、.m3u8保存/読み込み)
- フォルダ登録(登録フォルダの一覧からファイルをプレイリストへ展開)
- Whisper.netによるバックグラウンド自動文字起こし(ファイルを開いたら即再生開始、解析は裏で進行)
- **解析結果のストリーミング表示**(セグメント到着ごとに歌詞パネルへ逐次反映)
- **無音ハルシネーション対策**(VAD + フィルタパイプライン)
- **GPU自動フォールバック + モデル選択設定**
- ハッシュキーによるキャッシュで再解析を回避(原子書き込み)
- 字幕風の行(セグメント)単位ハイライト歌詞パネル、クリックでシーク
- ライセンス表記画面(LGPL/MIT表記)

含まない(将来拡張候補):
- ボーカル分離(BGM付き歌の認識精度が不足する場合に追加検討)
- 単語レベルのカラオケ風ハイライト
- 認識結果の手動テキスト修正
- クラウドAPI(OpenAI Whisper API等)への切替
- ライブラリのタグ付け・検索
- 解析途中結果のレジューム(途中終了したら再解析。graceful shutdownのみ実装)
- 複数音声トラックの選択UI(MVPは最初の音声トラック固定)
- キャッシュのGC/容量上限

```

    Cache/               … ハッシュ計算, キャッシュJSONの読み書き(原子書き込み)
```

## 処理の流れ

1. ユーザーがファイル/フォルダを開く、またはプレイリストから選択(ドラッグ&ドロップ対応)
2. `PlaybackService`(LibVLCSharp)が即座に再生開始
3. 並行して `CacheService` がファイルの合成ハッシュキーを計算し、キャッシュを探索
4. キャッシュ命中時はJSONを読み込むだけ。未命中時は解析キューへ投入し、`TranscriptionService` がバックグラウンドで処理:
   - 動画ファイルはffmpeg(LGPL)で音声トラックを抽出し、16kHz mono WAVへ変換(最初の音声トラック固定)
   - VAD(Silero VAD / whisper.cpp統合)で非音声区間をスキップ
   - Whisper.netで文字起こし。**セグメント到着ごとにイベントを発火**し、ハルシネーションフィルタ通過後にUIへ逐次反映
   - 完了後、結果をキャッシュJSONとして原子書き込み(.tmp→rename)
5. `LyricsViewModel` が再生位置に応じて該当行をハイライト・自動スクロール

## 解析キューと並行制御

- **Whisperの同時実行は常に1件**。解析要求はFIFOキューに積む
- 曲を切り替えても進行中の解析は**キャンセルせず完走**させ、キャッシュに保存する(次回同じ曲を開いたとき解析済みになる)。新しく開いた曲はキューの先頭に割り込ませる
- キュー上限は5件。超過したら最も古い未着手ジョブを破棄
- 同一ハッシュキーの解析が二重に走らないよう、キー単位のインフライト管理(`ConcurrentDictionary<key, Task>`)を行う
- 全パイプラインに `CancellationToken` を通す。アプリ終了時は解析をキャンセルし、一時WAVを削除する(graceful shutdown)

## 無音ハルシネーション対策

ASMRは無音・非音声区間(咀嚼音、タッピング等)が大半を占め、Whisperが無音部で「ご視聴ありがとうございました」等の定型句を繰り返し生成する既知問題に直撃する。Transcription層に以下のフィルタパイプラインを組み込む:

1. **VAD前処理**: whisper.cppのSilero VAD統合を有効化し、非音声区間をスキップ
2. **no_speech_probしきい値**: しきい値超過セグメントを破棄
3. **圧縮率(compression ratio)しきい値**: 繰り返しループを検出・破棄
4. **condition_on_previous_text無効化**: 繰り返し連鎖を抑制
5. **定型句フィルタ**: 既知のハルシネーション定型句(「ご視聴ありがとうございました」「チャンネル登録」等)のリストと照合して破棄

具体的なしきい値は実装時のスパイクで実サンプル(ささやきASMR・BGM付き歌)を使って調整する。

## キャッシュ設計

**ハッシュキー算出**

ファイル全体のフルハッシュは大きい動画で時間がかかるため、高速な合成キーを使用する。

```
key = SHA256( fileSize + head(1MB) + tail(1MB) )
```

ファイルのリネームやフォルダ移動があっても内容が同じであれば同一キーになる。既知の制約として記録しておく:
- タグ編集(ID3書き換え等)でハッシュが変わり再解析になる(許容)
- 先頭/末尾1MBとサイズが一致する別内容ファイルは理論上衝突するが実用上無視できる
- 再生中ファイルの読み取りは `FileShare.ReadWrite | FileShare.Read` で開く(LibVLCとの共有アクセス)

**保存場所とファイル名**

```
%AppData%\YomiagePlayer\cache\{key}-{model}.json
```

ファイル名にモデル名を含めることで、モデル切替時の再解析判定がファイル存在チェックだけで済む。

**キャッシュJSONスキーマ**

```json
{
  "version": 1,
  "sourceFileName": "song.mp3",
  "hashKey": "ab12cd34...",
  "model": "whisper-medium",
  "language": "ja",
  "durationSec": 245.3,
  "createdAt": "2026-07-08T10:00:00+09:00",
  "segments": [
    { "start": 12.34, "end": 15.10, "text": "夏の終わりに聞こえた声" },
    { "start": 15.10, "end": 18.20, "text": "遠くで揺れる風鈴の音" }
  ]
}
```

- `segments` はWhisperのセグメント単位(文/フレーズ)をハルシネーションフィルタ通過後に格納
- `sourceFileName` はデバッグ・表示用の参考情報(キー自体はハッシュ)

**書き込み・読み込み**

- 書き込みは `{key}-{model}.json.tmp` に書いてから `File.Move` でrename(原子的)。解析中のクラッシュで破損キャッシュが残らない
- 読み込み時は `version` とスキーマを検証し、不正なら破棄して再解析
- MVPでは「キャッシュがあれば常に使う」。明示的な再解析は右クリックメニューから手動実行可能

## 一時ファイル管理

- 抽出WAVは `%AppData%\YomiagePlayer\temp\` に置く(16kHz mono 16bitで約115MB/時間)
- 起動時に temp フォルダの残骸を掃除(前回クラッシュ対策)
- 解析完了・キャンセル時に該当WAVを削除

## UIレイアウト

```
┌─────────────────────────────────────────────────────┐
│ メニューバー: ファイル / 表示 / 設定                    │
├───────────┬───────────────────────────┬──────────────┤
│           │                           │              │
│ ライブラリ │   映像表示 / アルバムアート  │  歌詞パネル  │
│  (登録    │                           │  (字幕風・    │
│  フォルダ  │                           │  自動スクロール│
│  ツリー)  │                           │  現在行ハイライト)│
│           │                           │  [解析中 42%] │
│           ├───────────────────────────┤              │
│ プレイ    │  ▶ ⏸ ⏮ ⏭  ──●───── 🔊    │  クリックで   │
│ リスト    │  00:12 / 03:45  🔁 🔀     │  シーク       │
└───────────┴───────────────────────────┴──────────────┘
```

**主要インタラクション**

- 左ペイン上部「ライブラリ」: 登録フォルダの一覧(フォルダ追加/削除)。クリックで中の対応ファイルをプレイリストへ展開/追加
- 左ペイン下部「プレイリスト」: 現在の再生キュー。ドラッグ&ドロップで並び替え(GongSolutions.WPF.DragDrop利用を想定)、右クリックで削除・保存(.m3u8)/読み込み
- 中央: 動画ならVLCの映像、音声のみならファイル名表示(ジャケット画像はMVP外)。下部に標準的な再生コントロール
- 右ペイン「歌詞パネル」: セグメントを逐次リスト表示し、再生位置に応じて該当行をハイライト+自動スクロール。行クリックでその時刻へシーク
- ファイルを開く手段: メニュー「ファイルを開く」「フォルダを開く」、ドラッグ&ドロップ(※Airspace問題により映像領域上のD&Dは効かない可能性があるため、受付領域は実装スパイクの結果で決める)
- 設定画面: Whisperモデル選択(small/medium/large-v3-turbo)、使用ランタイム表示(CUDA/Vulkan/CPU)、ライセンス表記

**歌詞パネルの挙動詳細**

- **逐次表示**: 解析中もセグメント到着ごとに行を追加。進捗は「最後のセグメントend時刻 ÷ 総尺」で%表示
- **未解析区間へのシーク**: 解析済み区間より先へシークした場合は「この区間は解析中です」と表示
- **自動スクロールとの競合**: ユーザーが手動スクロールしたら自動スクロールを一時停止し、数秒無操作または「現在行に戻る」ボタンで復帰
- **無音区間**: どの行もハイライトしない(直前行の点灯は維持しない)

**歌詞パネルの状態遷移**

`Idle → (自動)Analyzing(逐次表示) → Ready` / `Analyzing → Failed(再解析ボタンで再度Analyzingへ)`

## エラーハンドリング

- ffmpeg抽出失敗(壊れたファイル/非対応コーデック/音声トラックなし): 歌詞パネルに「音声を抽出できませんでした」+再試行ボタンを表示。再生自体はLibVLCが対応していれば継続可能
- Whisper.net実行時例外(モデル未DL・メモリ不足等): 「文字起こしに失敗しました」を表示し詳細をログへ記録、再試行ボタンを提供
- モデルファイル未取得時: 初回起動時(またはモデル変更時)にダウンロード確認ダイアログを表示し、Hugging Faceのggml変換モデルを `%AppData%\YomiagePlayer\models\` に保存。SHAチェックサム検証を行う。オフライン時は「モデルがないため解析できません」と表示し再生のみ可能
- キャッシュJSON破損時: 読み込みを諦めて再解析する
- GPU初期化失敗: CUDA→Vulkan→CPUの順に自動フォールバックし、使用ランタイムを設定画面に表示

## 実装初期に行う技術スパイク

実装第1週に以下を検証し、結果次第で設計を微修正する:

1. **LibVLCSharp.WPFのAirspace問題**: `VideoView` 上でのドラッグ&ドロップ・UIオーバーレイの挙動を実機確認。問題があればD&D受付をプレイリストペインへ限定する等の回避策を決定
2. **Whisper実測**: 想定サンプル(ささやきASMR・BGM付き歌)で medium / large-v3-turbo の精度・処理速度を CPU/GPU で実測。ハルシネーションフィルタのしきい値を調整

## テスト方針

- 単体テスト(xUnit): ハッシュキー算出ロジック、キャッシュ読み書き(原子性・破損検証含む)、再生位置→現在セグメント検索(二分探索、無音区間は非ハイライト)、プレイリスト並び替えロジック、ハルシネーション定型句フィルタ、解析キュー(割り込み・上限・重複抑止)
- 手動/統合テスト: 実際の音声・動画サンプル(ASMR/BGM付き歌/動画/音声トラックなし動画/破損ファイル)でパイプライン一連(抽出→VAD→認識→フィルタ→キャッシュ→逐次表示)を通し確認。LibVLCSharpでの各フォーマット再生確認

## 未決事項(実装時に決定)

- 配布形態(zip配布/インストーラ/MSIX、自己完結型 or フレームワーク依存)とコード署名の要否
- 多重起動の扱い(シングルインスタンス化するか。二重起動時の同一キャッシュ同時書き込みは原子書き込みで実害を最小化)
- キャッシュJSONの非互換バージョン移行方針(MVPでは破棄→再解析で十分)
