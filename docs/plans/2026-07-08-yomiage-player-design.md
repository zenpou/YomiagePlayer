# YomiagePlayer 設計書

作成日: 2026-07-08

## 概要

Windows向けのメディアプレイヤー。ASMR・話し声・歌(BGMあり)などの音声/動画ファイルを解析し、日本語の歌詞・文章に書き起こして、再生に同期した字幕風表示を行う。一度解析した内容はキャッシュファイルに保存し、同じファイルの再解析を避ける。加えて、フォルダ登録・プレイリストなど基本的なメディアプレイヤー機能を備える。

## 技術スタック

- 実装言語/フレームワーク: C# / .NET 8, WPF
- 再生エンジン: LibVLCSharp(VLCエンジン) — mp3/mp4/mkv/flac/wavなど幅広いフォーマットをコーデック追加不要で再生
- 音声認識: Whisper.net(whisper.cppのC#バインディング)、デフォルトモデルは `medium`。Pythonランタイム不要でexe配布が容易
- 動画からの音声抽出・フォーマット変換: FFMpegCore(ffmpeg実行ファイルをバンドルまたは初回ダウンロード)。Whisper.net向けに16kHz monoのWAVへ変換

## MVPスコープ

含む:
- LibVLCSharpによる音声/動画再生 + 基本コントロール(再生/一時停止/前後/シーク/音量/リピート/シャッフル)
- プレイリスト(ドラッグ&ドロップ並び替え、.m3u8保存/読み込み)
- フォルダ登録(登録フォルダの一覧からファイルをプレイリストへ展開)
- Whisper.net(medium)によるバックグラウンド自動文字起こし(ファイルを開いたら即再生開始、解析は裏で進行)
- ハッシュキーによるキャッシュで再解析を回避
- 字幕風の行(セグメント)単位ハイライト歌詞パネル、クリックでシーク

含まない(将来拡張候補):
- ボーカル分離(BGM付き歌の認識精度が不足する場合に追加検討)
- 単語レベルのカラオケ風ハイライト
- 認識結果の手動テキスト修正
- クラウドAPI(OpenAI Whisper API等)への切替
- ライブラリのタグ付け・検索

## プロジェクト構成

```
YomiagePlayer/
  App/                  … WPFエントリポイント、DIコンテナ設定
  UI/                   … Views (MainWindow, PlaylistPane, LyricsPane, PlayerControls)

    Playback/           … LibVLCSharpをラップするPlaybackService
    Transcription/       … Whisper.net呼び出し, ffmpeg前処理, セグメント整形
    Cache/               … ハッシュ計算, キャッシュJSONの読み書き

  Models/               … TranscriptionResult, Segment, PlaylistItem, LibraryFolder
```

## 処理の流れ

1. ユーザーがファイル/フォルダを開く、またはプレイリストから選択(ドラッグ&ドロップ対応)
2. `PlaybackService`(LibVLCSharp)が即座に再生開始
3. 並行して `CacheService` がファイルの合成ハッシュキーを計算し、キャッシュを探索
4. キャッシュ命中時はJSONを読み込むだけ。未命中時はバックグラウンドで `TranscriptionService` を起動:
   - 動画ファイルはffmpegで音声トラックを抽出し、16kHz mono WAVへ変換
   - Whisper.net(medium)で文字起こしし、セグメント単位(開始/終了時刻+テキスト)を取得
   - 結果をキャッシュJSONとして保存
5. 解析完了を検知したら `LyricsViewModel` が字幕データを読み込み、再生位置に応じて該当行をハイライト・自動スクロール

## キャッシュ設計

**ハッシュキー算出**

ファイル全体のフルハッシュは大きい動画で時間がかかるため、高速な合成キーを使用する。

```
key = SHA256( fileSize + head(1MB) + tail(1MB) )
```

ファイルサイズと先頭1MB・末尾1MBの内容を組み合わせてハッシュ化する。ファイルのリネームやフォルダ移動があっても内容が同じであれば同一キーになる。

**保存場所**

```
%AppData%\YomiagePlayer\cache\{key}.json
```

**キャッシュJSONスキーマ**

```json
{
  "version": 1,
  "sourceFileName": "song.mp3",
  "hashKey": "ab12cd34...",
  "model": "whisper-medium",
  "language": "ja",
  "durationSec": 245.3,
  "createdAt": "2026-07-08T10:00:00+09:00",
  "segments": [
    { "start": 12.34, "end": 15.10, "text": "夏の終わりに聞こえた声" },
    { "start": 15.10, "end": 18.20, "text": "遠くで揺れる風鈴の音" }
  ]
}
```

- `segments` はWhisperのセグメント単位(文/フレーズ)をそのまま採用し、字幕風ハイライトに直結させる
- `sourceFileName` はデバッグ・表示用の参考情報(キー自体はハッシュ)
- `model` を保持しておき、将来モデルを切り替えた際の再解析判定に利用できるようにする

**キャッシュ探索ロジック**

1. ファイルを開いたら合成キーを計算(数十ms程度で完了)
2. `cache\{key}.json` が存在すれば即読み込みし解析をスキップ
3. 存在しなければバックグラウンド解析を起動し、完了後に同名で保存
4. MVPでは「キャッシュがあれば常に使う」というシンプルな挙動とし、明示的な再解析は右クリックメニュー等から手動実行できるようにする
5. キャッシュJSONが破損している場合は読み込みを諦めて再解析する

## UIレイアウト

```
┌─────────────────────────────────────────────────────┐
│ メニューバー: ファイル / 表示 / 設定                    │
├───────────┬───────────────────────────┬──────────────┤
│           │                           │              │
│ ライブラリ │   映像表示 / アルバムアート  │  歌詞パネル  │
│  (登録    │                           │  (字幕風・    │
│  フォルダ  │                           │  自動スクロール│
│  ツリー)  │                           │  現在行ハイライト)│
│           │                           │  [解析中...] │
│           ├───────────────────────────┤              │
│ プレイ    │  ▶ ⏸ ⏮ ⏭  ──●───── 🔊    │  クリックで   │
│ リスト    │  00:12 / 03:45  🔁 🔀     │  シーク       │
└───────────┴───────────────────────────┴──────────────┘
```

**主要インタラクション**

- 左ペイン上部「ライブラリ」: 登録フォルダの一覧(フォルダ追加/削除)。クリックで中の対応ファイルをプレイリストへ展開/追加
- 左ペイン下部「プレイリスト」: 現在の再生キュー。ドラッグ&ドロップで並び替え、右クリックで削除・保存(.m3u8)/読み込み
- 中央: 動画ならVLCの映像、音声のみならファイル名+ジャケット画像(あれば)を表示。下部に標準的な再生コントロール
- 右ペイン「歌詞パネル」: 解析済みセグメントをリスト表示し、再生位置に応じて該当行をハイライト+自動スクロール。行クリックでその時刻へシーク。解析中はスピナー+進捗表示、失敗時はエラー表示+再解析ボタン
- ファイルを開く手段: メニュー「ファイルを開く」「フォルダを開く」、ウィンドウへのドラッグ&ドロップ

**歌詞パネルの状態遷移**

`Idle → (自動)Analyzing → Ready` / `Analyzing → Failed(再解析ボタンで再度Analyzingへ)`

## エラーハンドリング

- ffmpeg抽出失敗(壊れたファイル/非対応コーデック): 歌詞パネルに「音声を抽出できませんでした」+再試行ボタンを表示。再生自体はLibVLCが対応していれば継続可能
- Whisper.net実行時例外(モデル未DL・メモリ不足等): 「文字起こしに失敗しました」を表示し詳細をログへ記録、再試行ボタンを提供
- モデルファイル(medium、約1.5GB)未取得時: 初回起動時にダウンロード確認ダイアログを表示し、`%AppData%\YomiagePlayer\models\` に保存
- キャッシュJSON破損時: 読み込みを諦めて再解析する

## 保存データ一覧

| データ | 保存先 |
|---|---|
| 文字起こしキャッシュ | `%AppData%\YomiagePlayer\cache\{hashKey}.json` |
| Whisperモデル | `%AppData%\YomiagePlayer\models\` |
| 登録フォルダ一覧・アプリ設定 | `%AppData%\YomiagePlayer\settings.json` |
| プレイリスト(明示保存時) | ユーザー指定パス `.m3u8` |
| ログ | `%AppData%\YomiagePlayer\logs\` |

## テスト方針

- 単体テスト(xUnit): ハッシュキー算出ロジック、キャッシュ読み書き、再生位置→現在セグメント検索(二分探索)、プレイリスト並び替えロジック
- 手動/統合テスト: 実際の音声・動画サンプル(ASMR/BGM付き歌/動画)でパイプライン一連(抽出→認識→キャッシュ→表示)を通し確認。LibVLCSharpでの各フォーマット再生確認
